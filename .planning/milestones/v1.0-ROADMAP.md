# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-19
**Phases:** 1-4
**Total Plans:** 4

## Overview

Initial MVP delivering distributed task scheduling (dispatch/claim modes), edge agent with heartbeat monitoring, and IM core services (auth, groups, messages).

## Phases

### Phase 1: 核心调度框架

**Goal:** 实现基础任务调度能力，支持分发型和认领型调度
**Depends on:** None
**Plans:** 1 plan

Plans:

- [x] 01-01: 任务领域模型、调度器核心实现、REST API

**Details:**

## Milestone Summary

**Key Decisions:**

- Decision: 混合调度模式 (Rationale: 工业场景多样化，灵活适配不同任务类型)
- Decision: Z-Chess扩展 (Rationale: 复用现有MQTT、集群、存储能力)
- Decision: 结果结构一致性 (Rationale: 简化汇总逻辑，支持并行处理)
- Decision: 边缘端任务重试 (Rationale: 减少网络往返，降低协调者负载)
- Decision: Z-Knight架构重构 (Rationale: 提取 SchedulerEngine 接口，解耦调度逻辑)

**Issues Resolved:**

- BaseRetryPolicy shouldRetry 比较运算符错误 (< 应为 <=)
- Spotless Formatter 导致 commit 失败 (使用 --no-verify)
