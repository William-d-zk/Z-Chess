## v1 Requirements (Complete)

### Distributed Computing (DIST)

- [x] **DIST-01**: 分发型调度 — 中心节点主动将任务拆分并分发到边缘节点执行 ✅ v1.0
- [x] **DIST-02**: 认领型调度 — 边缘节点从任务池主动认领任务，减少中心负载 ✅ v1.0
- [x] **DIST-03**: 分组分发 — 按边缘节点分组，定向分发任务到指定组 ✅ v1.0
- [x] **DIST-04**: 组内认领 — 组内边缘节点竞争认领，实现负载均衡 ✅ v1.0
- [x] **DIST-05**: 结果汇总 — 集群协调者收集并合并所有子任务结果 ✅ v1.0
- [x] **DIST-06**: 失败重试 — 任务失败在边缘端重试，节点失败由冗余算力重试 ✅ v1.0

### IM Services (IM)

- [x] **IM-01**: 用户认证 — 员工身份验证（内部SSO/LDAP集成） ✅ v1.0
- [x] **IM-02**: 群组聊天 — 群组对话、群管理（创建/解散/成员管理） ✅ v1.0
- [x] **IM-03**: 消息推送 — 在线消息实时推送，离线消息持久化 ✅ v1.0
- [ ] **IM-04**: 设备操作 — IM中可直接下发设备控制指令到边缘节点 (Phase 5)
- [ ] **IM-05**: 状态监控 — IM中可订阅设备状态，变更推送到对应会话 (Phase 5)
- [ ] **IM-06**: 数据订阅 — 聊天主题关联数据订阅，变换后自动推送到会话 (Phase 5)
- [ ] **IM-07**: 工作流提醒 — 任务状态变更、报表生成等事件通过IM提醒 (Phase 5)
- [ ] **IM-08**: 4/5G接入时SIM/eSIM绑定 — 设备SIM卡与边缘节点身份绑定 (Phase 5)

### Latency Requirements (LAT)

- [ ] **LAT-01**: 设备控制指令延迟 < 40ms（端到端，含网络传输） (Phase 6)
- [ ] **LAT-02**: AI模型推理延迟 10-30秒（视模型复杂度） (Phase 6)
- [ ] **LAT-03**: 数据分析报表延迟 12小时（批处理场景） (Phase 6)
