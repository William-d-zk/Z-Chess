# Features Research: 边缘算力调度与IM平台

## 功能分类

### Table Stakes（必须具备，否则用户会离开）

#### 分布式计算核心

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| **任务分发** | 中心节点将任务拆分并分发到边缘 | 高 |
| **任务认领** | 边缘节点主动从任务池认领任务 | 高 |
| **结果汇总** | 协调者收集并合并子任务结果 | 中 |
| **失败重试** | 任务失败边缘端重试，节点失败冗余算力重试 | 高 |
| **任务状态跟踪** | 全链路任务状态可视化管理 | 中 |

#### IM基础能力

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| **用户认证** | 员工身份验证（内部SSO/LDAP） | 中 |
| **一对一消息** | 私聊功能 | 低 |
| **群组聊天** | 群组对话、群管理 | 中 |
| **消息推送** | 在线消息实时推送，离线消息存储 | 中 |

### Differentiators（竞争优势）

#### 智能调度

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| **分组分发** | 按边缘节点分组，定向分发任务 | 高 |
| **组内认领** | 组内竞争认领负载均衡 | 高 |
| **算力感知调度** | 根据节点CPU/内存/GPU动态分配 | 高 |
| **任务亲和性** | 数据局部性优化，减少传输 | 中 |

#### 设备与IM联动

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| **IM设备操作** | 在IM中直接下发设备控制指令 | 高 |
| **IM设备监控** | 订阅设备状态推送到IM会话 | 中 |
| **数据订阅推送** | 聊天主题关联数据自动推送 | 高 |
| **智能体协作** | IM中AI智能体参与讨论和任务 | 高 |

#### 高级特性

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| **工作流编排** | 多个任务按依赖关系编排执行 | 高 |
| **任务优先级** | 紧急任务抢占调度 | 中 |
| **多租户隔离** | 不同组织数据隔离 | 高 |
| **边缘自治** | 断网情况下边缘节点独立运行 | 极高 |

### Anti-Features（明确不构建）

| 功能 | 原因 |
|------|------|
| 公有云部署 | 专注内部工业场景 |
| 通用任务市场 | 专注工业边缘，非通用计算 |
| 视频/流媒体处理 | 带宽消耗大，非核心需求 |
| 移动端IM | 内部员工PC为主 |
| 第三方IM集成 | 自建控制力更强 |

## 功能依赖关系

```
设备接入 (Z-Pawn)
    ↓
任务调度层
    ├── 分发型任务 ──────────┐
    ├── 认领型任务 ──────────┤
    ├── 分组分发任务 ─────────┼──→ 边缘Agent执行
    └── 组内认领任务 ────────┘
    ↓
结果汇总层 (Z-Knight RAFT)
    ↓
IM服务层
    ├── 设备操作 ──────────→ 设备控制指令
    ├── 状态监控 ──────────→ MQTT订阅
    └── 数据推送 ──────────→ 消息推送
```

## 复杂度评估

- **简单**：一对一消息、用户认证、消息推送
- **中等**：群组聊天、结果汇总、状态跟踪
- **复杂**：四种调度模式、设备IM联动、智能体协作
- **极高**：边缘自治、多租户隔离

- Differentiators功能：★★★★☆ 需求明确
