# Stack Research: 边缘算力调度与IM平台

## 标准技术栈建议

### 1. 任务调度层

**调度框架选择：**

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **自研调度器** | 工业边缘场景 | 完全可控，针对性优化 | 开发成本高 |
| Kubernetes + KEDA | 云边协同 | 生态成熟，伸缩能力强 | 资源占用较高 |
| Apache YARN | 离线批处理 | Hadoop生态成熟 | 实时性较弱 |
| HashiCorp Nomad | 轻量级调度 | 资源占用低 | 边缘场景案例少 |

**推荐：基于Z-Knight RAFT集群自研调度器**
- 契合Z-Chess已有架构
- 针对边缘设备的特殊约束（低功耗、低带宽、间歇连接）
- 支持四种调度模式：分发型、认领型、分组分发、组内认领

### 2. 边缘计算运行时

**边缘Agent要求：**
- 轻量级（<50MB内存占用）
- 支持容器化部署（Docker/Containerd）
- 任务子集执行环境隔离
- 心跳上报 + 任务状态同步

**推荐技术：**
- 自研Go-based Edge Agent（内存占用低，支持交叉编译）
- 或使用 WasmEdge（WebAssembly运行时，适合多语言任务）

### 3. 消息传递与服务发现

**Z-Chess已有MQTT层可复用**，扩展点：
- MQTT Topic设计支持任务分发/认领
- 使用MQTT v5.0共享订阅实现分组分发
- 发现服务利用Z-Knight RAFT集群

**IM消息服务：**
| 方案 | 推荐度 | 说明 |
|------|--------|------|
| 自研WebSocket服务 | ★★★★ | 与Z-Chess架构统一，控制力强 |
| Go-socket.io | ★★★ | 简单实时消息 |
| Centrifugo | ★★★ | 成熟Proxies服务端 |
| Matrix协议 | ★★★ | 开源IM协议，自建服务器 |

**推荐：自研WebSocket服务，复用Z-Queen的AIO能力**

### 4. 数据存储

**Z-Chess已有：**
- PostgreSQL（Z-Rook）：关系数据、任务状态、用户数据
- EhCache：缓存层

**新增需求：**
- 边缘节点元数据（设备能力、算力、状态）
- 任务执行历史（用于分析）
- IM消息持久化

**推荐架构：**
- PostgreSQL + TimescaleDB（时序数据）
- 或 Cassandra（大规模消息存储）

### 5. 延迟保障

| 延迟级别 | 实现方案 |
|---------|---------|
| <40ms 控制指令 | MQTT QoS1/2 + 边缘就近执行 + Z-Queen AIO |
| 10-30s AI推理 | 任务预标记优先级 + 边缘GPU/NPU算力预留 |
| 12h 报表 | 批处理 + 定时任务 + 分片聚合 |

### 6. 完整技术栈

```
┌─────────────────────────────────────────────────────────────┐
│                        Z-Chess 扩展                          │
├─────────────────────────────────────────────────────────────┤
│  网关层  │  Z-Arena (扩展路由)  │  Z-Player (API)           │
├─────────────────────────────────────────────────────────────┤
│  IM服务  │  自研WebSocket服务    │  群组/文件/提醒           │
├─────────────────────────────────────────────────────────────┤
│  调度层  │  Z-Knight (RAFT)     │  自研调度器               │
├─────────────────────────────────────────────────────────────┤
│  边缘层  │  Edge Agent (Go)     │  WasmEdge (可选)          │
├─────────────────────────────────────────────────────────────┤
│  存储层  │  Z-Rook (PostgreSQL) │  EhCache + TimescaleDB   │
├─────────────────────────────────────────────────────────────┤
│  通信层  │  Z-Queen (AIO)       │  MQTT v5.0 (复用)         │
└─────────────────────────────────────────────────────────────┘
```

### 7. 不推荐的技术

| 技术 | 原因 |
|------|------|
| Kafka | 延迟高，不适合实时边缘场景 |
| Redis Cluster | 边缘环境部署复杂 |
| gRPC | 边缘设备兼容性问题，穿透NAT麻烦 |
| Docker Swarm | 边缘场景生态弱于K8s |
