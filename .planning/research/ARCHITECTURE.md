## 核心组件边界

### 1. 任务调度器 (Task Scheduler)

**职责：**
- 接收计算任务请求
- 将任务拆分为子任务
- 根据调度模式分发/认领
- 跟踪任务生命周期

**数据流：**
```
用户/系统 → 任务请求 → 任务拆分 → 调度决策 → 子任务分发
                                              ↓
结果汇总 ← 完成通知 ← 执行完成 ← 边缘节点执行
```

**关键接口：**
```java
interface TaskScheduler {
    // 分发型：主动推送到边缘
    void dispatch(Task task, List<EdgeNode> targets);
    
    // 认领型：边缘主动认领
    List<SubTask> claimTasks(EdgeNode node, int maxCount);
    
    // 分组分发：按组定向分发
    void dispatchToGroup(Task task, String groupId);
    
    // 组内认领：组内竞争
    List<SubTask> groupClaimTasks(String groupId, int maxCount);
}
```

### 2. 结果汇总器 (Result Aggregator)

**职责：**
- 收集子任务结果
- 检测部分失败
- 触发重试或冗余计算
- 合并同构结果

**状态机：**
```
PENDING → RUNNING → PARTIAL_COMPLETE → COMPLETE
                ↘→ FAILED → RETRY → RUNNING
```

### 3. 边缘Agent (Edge Agent)

**职责：**
- 向调度器注册算力
- 接收/认领子任务
- 在本地执行计算
- 报告执行结果

**部署架构：**
```
┌─────────────────┐
│   边缘设备       │
│  ┌───────────┐  │
│  │Edge Agent │  │
│  └─────┬─────┘  │
│        │        │
│  ┌─────┴─────┐   │
│  │任务执行器 │   │
│  │(Docker)  │   │
│  └───────────┘   │
│        │        │
│  ┌─────┴─────┐   │
│  │本地存储   │   │
│  └───────────┘   │
└─────────────────┘
```

### 4. IM服务 (IM Service)

**职责：**
- 用户认证与会话管理
- 消息收发（群组/私聊）
- 设备操作命令转发
- 数据订阅与推送

**与调度层交互：**
```
IM消息 → 解析意图 → 设备操作API → 任务调度器
                                    ↓
IM订阅 ← 状态更新 ← MQTT订阅 ← 设备/计算结果
```

## 数据流设计

### 实时控制流（<40ms）

```
用户IM → 设备操作指令 → Z-Player API → Z-Knight调度
                                               ↓
                              边缘节点执行 ← MQTT分发
                                               ↓
                              结果确认 → IM推送
```

### AI推理流（10-30s）

```
任务提交 → 任务拆分(100子任务) → 分组分发
                                          ↓
                    边缘节点认领 → 执行 → 结果上报
                                          ↓
                           结果汇总(100/100) → 推送结果
```

### 批处理流（12h）

```
定时触发 → 任务入队 → 调度器批量分发
                                      ↓
                    边缘节点执行 → 结果存储
                                      ↓
                    12h后汇总 → 生成报表 → IM通知
```

## 关键架构决策点

| 决策 | 选项 | 推荐 | 原因 |
|------|------|------|------|
| 任务状态存储 | Z-Knight RAFT | PostgreSQL | 需要持久化，RAFT内存有限 |
| 消息传递 | MQTT + WebSocket | MQTT用于设备，WebSocket用于IM | 复用Z-Chess MQTT |
| 边缘隔离 | Docker容器 | 进程级隔离 | 资源占用考虑 |
| 结果合并 | 中心汇总 | 流式合并 | 同构结果简单汇总 |
